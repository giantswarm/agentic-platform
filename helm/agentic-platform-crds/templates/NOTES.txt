This chart ships the CustomResourceDefinitions for the agentic platform:

  agentgateway-crds  → agentgatewayparameters.agentgateway.dev
                       agentgatewaypolicies.agentgateway.dev
                       agentgatewaybackends.agentgateway.dev
  kagent-crds        → agents/agentharnesses/modelconfigs/mcpservers/
                       remotemcpservers/memories/toolservers/sandboxagents.kagent.dev
  agent-sandbox-crds → sandboxes/sandboxtemplates/sandboxclaims/sandboxwarmpools.agents.x-k8s.io

  NOTE: muster's CRDs (mcpservers/workflows.muster.giantswarm.io) are NOT shipped
  here — muster owns them in its own app chart (app-owned CRDs, upgraded via Flux
  CreateReplace).

INSTALL ORDER: install (or upgrade) this CRDs release BEFORE the companion
agentic-platform release. The platform chart renders CRs (AgentgatewayParameters,
Gateway, kagent CRs) that require these CRDs to already exist and be Established.

Verify the CRDs are registered and Established before installing the platform:

  kubectl get crd \
    agentgatewayparameters.agentgateway.dev \
    agentgatewaypolicies.agentgateway.dev \
    agentgatewaybackends.agentgateway.dev \
    agents.kagent.dev

  kubectl wait --for=condition=Established crd/agentgatewayparameters.agentgateway.dev

Gateway API CRDs (gateways/httproutes/gatewayclasses.gateway.networking.k8s.io)
remain a cluster prerequisite and are NOT shipped by this chart.

UNINSTALL WARNING (mixed delete behaviour):
  - agent-sandbox CRDs (*.agents.x-k8s.io) carry `helm.sh/resource-policy: keep`
    — they and their CRs survive `helm uninstall {{ .Release.Name }}`.
  - agentgateway CRDs (*.agentgateway.dev) and kagent CRDs (*.kagent.dev) do NOT
    carry the keep policy (upstream exposes no knob to set it). `helm uninstall
    {{ .Release.Name }}` DELETES them and cascades to every agentgateway / kagent
    CR cluster-wide. Treat uninstalling this release as destructive for those CRs.
    Tracked upstream — see README.md.
